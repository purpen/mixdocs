新增分类¶

商家自定义分类

接口说明¶

  • API接口请求地址：/categories
  • API接口请求方法：POST
  • API接口用户授权：token

请求参数¶

名称 类型 是否必须 默认值 描述说明
pid Number 可选 0 父级ID
name String 必填   分类名称
description String 可选   分类描述
sort_order Number 可选 0 排序，数字越大越靠前

请求示例¶

请求参数必须以 JSON 格式形式，服务端才会获取请求的数据。 

{"name": "新分类名称", "sort_order": 5, "description": "新分类描述说明"}

返回示例¶

JSON数据格式:

{
  "data": {
    "cover": "http://kg.erp.taihuoniao.com/static/img/default-logo-180x180.png",
    "description": "新分类描述说明",
    "id": 21,
    "name": "新分类名称",
    "pid": 0,
    "sort_order": 5,
    "status": 1
  },
  "status": {
    "code": 201,
    "message": "All created."
  }
}

  • API接口请求地址：/categories/<id>
  • API接口请求方法：PUT
  • API接口用户授权：token

请求参数¶

名称 类型 是否必须 默认值 描述说明
pid Number 可选 0 父级ID
name String 必填   分类名称
description String 可选   分类描述
sort_order Number 可选 1 排序，数字越大越靠前

参数要求参考 新增分类 示例

     "description": "新分类描述说明",
     "id": 21,
     "name": "新分类名称-更新",
     "pid": 0,
     "sort_order": 5,
     "status": 1
   },
   "status": {
     "code": 200,
     "message": "Ok all right."
   }
 }

删除分类¶

删除商家自定义分类

接口说明¶

  • API接口请求地址：/categories/<id>
  • API接口请求方法：DELETE
  • API接口用户授权：token

请求参数¶

名称 类型 是否必须 默认值 描述说明
id Number 必须   分类ID

返回示例¶

JSON数据格式:

请求 正确 返回结果：

{
    "status": {
        "code": 204,
        "message": "All deleted."
    },
    "success": true
}

请求 失败 返回结果：

{
    "status": {
        "code": 403,
        "message": "该分类包含商品, 不能删除！"
    },
    "success": false
}

qk String 可选   搜索关键词
min_price Number 可选   价格区间： 最小价格

sort_type Number 可选   排序方式 0=默认排序, 1=综合排序, 2=价格由低至高, 3=价格由高至低
is_free_postage Number 可选 0 是否包邮: 0=全部, 1=包邮
